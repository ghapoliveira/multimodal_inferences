# Experiment 2 — EEG study (sentence–emoji bridging)

Rapid Serial Visual Presentation (RSVP) EEG experiment on bridging inferences between text and emojis. The 120 target items are the subset selected in Experiment 1 (`../exp1_norming`), interleaved with 120 fillers and presented in E-Prime while EEG was recorded. This directory holds the experiment, the preprocessing pipeline, and two statistical analyses: a Spatio-temporal Cluster-based Permutation Test and a regression-ERP (rERP). It also contains all the plots and pictures presented in the Dissertation (Chapter 4).

Pipeline can follow two alternative paths by switching parameters in 3_segmenting_epochs.ipynb and/or 4_averaging.ipynb. By switching "IS_BASELINE_CHECK=True" in both scripts, the time before the trigger is epoched and averaged for later analysis (replacing the standard epoching); by switching "ALT_PIPELINE=True" in 4_averaging.ipynb, the items mentioned in the Dissertation (2, 61 and 62) are removed. 3_segmenting_epochs.ipynb attaches item metadata to the EEG data so removing items (or item statistics) is possible later.

Every stochastic process was made reproducible by setting random_state = 42. However, Autoreject needs special attention: it silently ignores the parameter if n_jobs is set to \> 1. Parallel processing is inherently non-deterministic: whichever partitioned process ends first influences the outcome randomly and the result is always slightly different, despite random_state = 42. All analyses were, therefore, run with n_jobs = 1.

## The three pipeline configs

`run_pipeline.py` runs each participant through three variants (set in `pipeline_jobs`), all sharing steps 1–2:

- **standard** — the main analysis.
- **baseline** — a baseline-check variant (pre-stimulus window −1.2–0 s, baseline (−1.2,−1)).
- **alt_pipeline** — identical to standard but drops items `ITEMS_TO_DROP = [2, 61, 62]` (a robustness check).

Outputs land in each participant folder: standard in the root, baseline in `baseline_check/`, alt in `alternative_pipeline/`. Every participant has its notebook frozen for inspection.

## Reproducing

**1. Environment.** EEG work spans three languages:

``` bash
# Python (preprocessing + cluster + grand averages)
conda env create -f environment.yml          # or: pip install -r requirements.txt
# R (behavioral + rERP plotting) — see requirements_R.txt for versions.
# Julia (rERP fitting) — see statistics/rERP/ for the Project.toml
```

**2. Lists**:

``` bash
python create_lists_eeg.py     # reads ../exp1_norming + materials/, writes results/
```

The presented lists were generated unseeded and finalized for E-Prime; the canonical record is `results/eeg_lists_as_used.xlsx`. Re-running reproduces the *method*, not that exact file.

**3. Preprocessing** (from `preprocessing/`):

``` bash
python run_pipeline.py
```

Reads raw EEG from `data/eeg/participant_N/`, writes per-participant evokeds. It prints an `INCOMPLETE PARTICIPANTS` summary at the end — if any participant appears there, that subject did not finish and must be re-run before analysis.

**4. Analysis** (from `analysis/`):

``` bash
jupyter run grand_averages.ipynb                      # grand averages + diff waves
jupyter run statistics/cluster_based/cluster_based.ipynb   # permutation test (seed=42)
Rscript behavioral_stats.r                            # comprehension accuracy
# rERP: see statistics/rERP/ (prepare_rERP.ipynb -> rERP.jl -> plot_rERP.r)
```

## Notes

> Reproducibility: ICA and AutoReject use `random_state=42`; the cluster permutation test uses `seed=42` with 50,000 permutations. The EOG proxies for ICA are the frontal channels Fp1/Fp2/F7/F8 (no dedicated EOG electrodes), with a threshold that steps down until ≥2 components are flagged (Aaron Newman's procedure — https://neuraldatascience.io/). Subject exclusions live in `config.py` (`BAD_SUBJ`), and the loaders assert the expected participant count so a missing or duplicated subject is a hard error, not a silently shifted average.

> create_lists_eeg.py: the Fillers had a tag inconsistency (the tag ESC — school — was sometimes written ESCOLA). This did not cause any major ordering problems, but this was fixed for the published script.

> Participants: Participants 10 and 29 started late, so they contribute fewer epochs. Additionally, Participant 3, who was removed during preprocessing, contributes to the E-Prime stats (so the count is 29, and not 30).

> AI assistance: The development of the scripts in this repository was AI-assisted (Google Gemini and Claude).
